算法基本概念

# 算法基础

著名的图灵奖获得者Donald E. Knuth曾经说:"Computer science is the study of algorithm"，也就是计算科学就是研究算法的科学。这门学科主要的学习方法是通过经典算法的学习，积累经验，触类旁通，举一反三。

笔记

算法设计这门课学的是发现解决问题的算法，同时证明算法是正确、有效和能被接受的。不仅仅是找到答案，更要证明答案。通过例子来学习算法的思想，然后搬到其他问题中去解决其他问题。

# 什么是算法？(基本概念)

算法是解决某个问题的一系列运算或操作。编译器和操作系统就是典型的例子。算法问题主要包括输入和输出

  • 输入:明确了算法能够接受的合法输入
  • 输出:明确对于一组合法的输入，相应所得到的结果是什么

数学公式:output=f(input)output = f(input)output=f(input)，其中f(⋅)f(\cdot)f(⋅)就表示算法。

# 伪代码

伪代码是一种能够将标准运行的程序通过简介明了的语言进行描述，其特点是保持了程序原本主要的结构，类似C语言等。语法规则如下:

  • 赋值:←\larr←
  • 分支:if...then...else
  • 循环:while,for,repeat...until
  • 输出:return
  • 调用:写函数名
  • 注释: //...

特别注意！

需要特别注意的是，伪代码不能照搬原代码，需要进行部分精简，不需要具体的实现流程，只需要写出算法思路流程，分为哪几步，不用详细写这几步的实现过程，只需要使用自然语言或者相应的函数替代就行。忽略数据结构、模块、异常处理等。忽略变量的说明。

# 例子

Insert_Sort: 输入: nnn个元素的数组AAA; 输出: 按非降序排列的数组AAA;

for i <- 2 to n
  x <- A[i]
  j <- i - 1
  while (j > 0) and (x < A[j])
    A[j +  1] <- A[j]
    j <- j - 1
  A[j + 1] <- x
return A

# 算法复杂度

# 复杂度定义

算法的复杂度实际上是对算法执行次数的范围做一个估计。描述了一个算法执行次数至多和至少有多少，衡量一个算法的效率。描述的时候用到了高等数学中界的概念，一般有两种描述方式:

先设有任意正数K1K_1K1​，使得有f(n)≤K1f(n)\le K_1f(n)≤K1​，则称为f(n)f(n)f(n)的上界，记作f(n)=O(K1)f(n)=O(K_1)f(n)=O(K1​)，其中K1=cg(n)K_1=cg(n)K1​=cg(n)且存在正数ccc和n0n_0n0​，对一切的n≥n0n\ge n_0n≥n0​都成立。

同理，若任意正数K2K_2K2​，使得有f(n)≥K2f(n)\ge K_2f(n)≥K2​，则称为f(n)f(n)f(n)的下界，记作f(n)=Ω(K2)f(n)=\Omega(K_2)f(n)=Ω(K2​)，其中K2=cg(n)K_2=cg(n)K2​=cg(n)且存在正数ccc和n0n_0n0​，对一切的n≥n0n\ge n_0n≥n0​都成立。

如果，f(n)=O(g(n))f(n)=O(g(n))f(n)=O(g(n))且f(n)=Ω(g(n))f(n)=\Omega(g(n))f(n)=Ω(g(n))，则记作f(n)=Θ(g(n))f(n)=\Theta(g(n))f(n)=Θ(g(n)).

O(g(n))与Ω(g(n))的含义

  • O(g(n))O(g(n))O(g(n))表示该算法至多执行g(n)g(n)g(n)次（最多）
  • Ω(g(n))\Omega(g(n))Ω(g(n))表示该算法至少执行g(n)g(n)g(n)次（最少）

本质上算法复杂度会这样表示是因为简化表示，比如现在有一个程序的复杂度为nlog⁡n+n−2n\log n + n - 2nlogn+n−2，如果直接表示的话就是f(n)=nlog⁡n+n−2f(n)=n\log n+n-2f(n)=nlogn+n−2，但这样表示不够方便，而且有很多不必要的元素，因为算法复杂度其实不需要一个很精确的表示，因为就算你表示得很精确又不能说明什么，何必多此一举；而且在某个区间上来说，nlog⁡n≥nn\log n \ge nnlogn≥n那我既然都不用表示那么完整了，何必又要再写nnn和222？直接用最大的那个表示出来就可以了。所以就可以写成O(nlog⁡n)O(n\log n)O(nlogn).

  1. 不需要精确表示
  2. 不需要精确表示就可以对式子进行化简，留下最大那个
  3. 为什么留下最大那个，因为他起主导作用

  1. 最坏情况下时间复杂度: 算法求解输入规模为nnn的算法所需要的最长时间W(n)W(n)W(n)
  2. 平均情况下时间复杂度: 算法求解输入规模为nnn的算法所需要的平均时间A(n)A(n)A(n)

A(n)=∑I∈StIpIA(n)=\sum_{I\in S}t_Ip_I A(n)=I∈S∑​tI​pI​

其中，SSS:实例集, tIt_ItI​:实例I基本操作次数, pIp_IpI​: I的概率

笔记

复杂度表示的是针对问题选择基本运算，将基本运算的次数(输入规模)为自变量的函数。

  • 也就是说，复杂度中的nnn表示算法的输入规模，经过f(n)f(n)f(n)次运算之后消耗的时间。本质上时间复杂度就是一个表示操作次数的边界的函数。
